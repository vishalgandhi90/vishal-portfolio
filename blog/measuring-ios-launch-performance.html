Measuring iOS App Launch Performance with LaunchProfiler

How a simple milestone-based profiling library helped us identify and fix the bottlenecks that were slowing down our app's startup by 30%.

Swift Open Source Performance Profiling

Why Launch Time Matters

App launch time directly impacts user retention. Studies show that 25% of users abandon an app if it takes longer than 3 seconds to load. For a delivery app with 100K+ daily users, every millisecond of startup time counts.

We needed a way to consistently measure launch performance across builds, identify which initialization steps were the bottlenecks, and track improvements over time. Instruments is great for deep dives, but we needed something lightweight that could run in debug builds and CI.

Basic Usage

LaunchProfiler uses a milestone-based approach. You mark key points during your app's startup, and the profiler measures the time between them.

// AppDelegate.swift
func application(
    _ application: UIApplication,
    didFinishLaunchingWithOptions launchOptions: ...
) -> Bool {

    LaunchProfiler.shared.start()

    setupDependencies()
    LaunchProfiler.shared.mark("Dependencies")

    setupNetworking()
    LaunchProfiler.shared.mark("Networking")

    setupUI()
    LaunchProfiler.shared.mark("UI Setup")

    loadCachedData()
    LaunchProfiler.shared.mark("Cache")

    LaunchProfiler.shared.finish()
    return true
}

Console Output

When the profiler finishes, it prints a formatted report showing each milestone's duration and a classification indicator.

========================================
  Launch Profile Report
========================================
  Dependencies    120ms  ✅ Good
  Networking       45ms  ✅ Good
  UI Setup        380ms  ⚠️ Acceptable
  Cache           210ms  ⚠️ Acceptable
----------------------------------------
  Total           755ms  ✅ Good
  Slowest     UI Setup (380ms)
========================================

The classification (Good/Acceptable/Slow) is based on configurable thresholds, making it easy to spot regressions at a glance.

Configurable Thresholds

Every app has different performance budgets. LaunchProfiler lets you define what "good" means for your specific case.

let config = LaunchProfilerConfiguration(
    isEnabled: true,
    thresholds: LaunchThreshold(
        good: 0.5,       // Under 500ms = good
        acceptable: 1.0,  // Under 1s = acceptable
        slow: 2.0         // Over 2s = slow
    ),
    reportingMode: .console,
    isEnabledInRelease: false
)

LaunchProfiler.shared.configure(config)

Thread Safety

Launch profiling often spans multiple threads. The dependency injection container might initialize on a background queue, while UI setup happens on main. LaunchProfiler uses NSLock internally to ensure thread-safe milestone recording without the overhead of actors (which would require async context at call sites).

public final class LaunchProfiler {
    private let lock = NSLock()
    private var milestones: [Milestone] = []

    public func mark(_ name: String) {
        lock.lock()
        defer { lock.unlock() }
        milestones.append(
            Milestone(
                name: name,
                timestamp: CFAbsoluteTimeGetCurrent()
            )
        )
    }
}

I chose NSLock over actors because mark() is called from synchronous contexts like application(_:didFinishLaunchingWithOptions:) where adding await would be invasive and impractical.

Custom Reporting with Delegates

The console reporter is great for development, but in production you might want to send launch metrics to your analytics backend. The LaunchProfilerDelegate protocol lets you hook into the reporting pipeline.

class AnalyticsReporter: LaunchProfilerDelegate {
    func launchProfiler(
        didFinishWithReport report: LaunchReport
    ) {
        Analytics.track(
            "app_launch",
            properties: [
                "total_ms": report.totalDuration * 1000,
                "slowest": report.slowestMilestone?.name,
                "classification": report.classification.rawValue
            ]
        )
    }
}

LaunchProfiler.shared.delegate = AnalyticsReporter()

Real Impact

When we integrated LaunchProfiler into our delivery app, we discovered that:

  • Dependency injection was taking 320ms because we were eagerly resolving services that weren't needed at launch.
  • Feature flag fetching was blocking the main thread for 180ms. Moving it to a background queue with a cached fallback cut this to 5ms.
  • Image cache warming was loading 50+ category icons synchronously. Lazy loading reduced this from 250ms to 15ms.

Total result: launch time went from 1.2 seconds to 680ms, a 43% improvement. And because the profiler runs on every debug build, we catch regressions immediately.

Installation

Add LaunchProfiler to your project via Swift Package Manager.

// Package.swift dependency
.package(
    url: "https://github.com/vishalgandhi90/LaunchProfiler",
    from: "1.0.0"
)
